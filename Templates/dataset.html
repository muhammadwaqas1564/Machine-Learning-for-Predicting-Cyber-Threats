Android Malware Analysis
Dataset

The Android Malware Analysis dataset comprises 17,341 Android app samples, providing a large and robust resource for studying Android malware. This dataset offers valuable insights into various malicious app behaviors, code patterns, and distribution techniques. It has been gathered from a variety of reputable sources, ensuring diversity in the types of malware included. Researchers and security professionals can leverage this dataset for various tasks, such as training machine learning models for automated malware detection, analyzing malware behavior, and studying the evolution of Android malware threats.

DIVERSE CATEGORIES

The dataset includes five distinct app categories with a clear breakdown of malicious behaviors. These categories provide insights into different attack vectors, helping researchers understand malware diversity:

Adware

Adware bombards users with unwanted advertisements, often in the form of pop-ups or overlay ads. These apps often masquerade within legitimate applications, remaining dormant until triggered. Adware can lead to data breaches by stealing user information or performing unauthorized actions, such as clicking ads without user consent. Examples in the dataset include adware disguised as utility apps, such as battery savers or phone cleaners.

Banking Malware

Banking Malware targets banking apps or users accessing financial platforms. These malicious apps employ techniques such as phishing-style fake login pages to harvest credentials or keylogging to capture sensitive details. Advanced versions use real-time overlays, tricking users into entering their data while appearing to interact with a legitimate app. The dataset contains examples of fake apps impersonating well-known banks to steal login credentials.

SMS Malware

SMS Malware exploits SMS permissions to perform malicious actions. These include sending premium-rate texts without user knowledge, stealing verification codes used for two-factor authentication, or spreading malware via malicious links in SMS messages. Some samples are stealthy, intercepting or deleting messages to remain undetected. The dataset includes SMS Trojans masquerading as messaging apps.

Riskware

Riskware refers to legitimate apps with vulnerabilities that attackers can exploit to cause harm. These apps, while not inherently malicious, can be used to escalate privileges, leak sensitive data, or install additional malware if their weaknesses are exploited. Examples in the dataset include apps with weak encryption practices or poor permission management.

Benign

Benign apps serve as a baseline for malware detection tasks. These non-malicious apps are verified using tools like VirusTotal to ensure they are safe. They include everyday applications such as calculators, weather apps, and educational tools, which perform no harmful activities and are included in the dataset for comparative analysis.

Balancing the Dataset

To ensure fairness in training machine learning models, the dataset was balanced effectively. This ensures robust and unbiased performance in predictive tasks.

1. Even Category Distribution

Categories were adjusted so that each class has equal representation, minimizing bias and ensuring fairness in model training.

2. Shuffling

Random shuffling was applied to ensure that training and testing subsets are not biased toward a specific pattern or category.

3. Significance

Balancing is crucial for classification models to avoid overfitting on dominant categories (e.g., SMS malware), ensuring equal importance is given to all classes.

PUBLIC AVALABILITY & CITATION

The CICMalDroid 2020 dataset, available through the Canadian Institute for Cybersecurity (CIC) website, is a valuable resource for studying Android malware. Researchers can access it by registering, agreeing to terms of use, and citing it in their work. The dataset includes APK files, network logs, and CSV files, enabling static and dynamic analysis, as well as network behavior studies. Accompanied by documentation, it supports integration into machine learning workflows. By providing diverse malware samples, the dataset aids in detecting, classifying, and mitigating threats, fostering collaboration and innovation in Android security research.
